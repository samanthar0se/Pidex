# 49 — Complete soak, daily-driver, and release promotion

**What to build:** Assemble traced release evidence that Pidex can ship as the developer's daily Pi interface: sustained automated operation, real desktop/mobile use, supported install/upgrade/recovery paths, and a defect bar with no unresolved critical workflow or authority failures.

**Blocked by:** 48 — Complete the deterministic fault and recovery campaign

**Status:** ready-for-agent

- [ ] A 72-hour automated soak at each applicable capacity tier completes with zero invariant violation, daemon crash, stuck accepted work, incorrect Client convergence, unbounded queue growth, or resource-limit failure.
- [ ] After soak returns to equivalent quiescence, handle/memory resource gates remain satisfied with recorded diagnostic evidence.
- [ ] A seven-day real daily-driver trial performs core Pi work from desktop and mobile Pidex Clients without returning to another Pi UI for a core Session action.
- [ ] The daily-driver trial loses or duplicates no accepted prompt, steering, follow-up, Interaction Response, Stop, Timeline settlement, or maintenance operation and has no Severity 1/2 defect.
- [ ] Promotion exercises clean install, upgrade from every supported prior release, pre-acceptance rollback, uninstall/reinstall with data preservation, required browser/capacity matrix, security automation, Pi contracts, fault/recovery suites, soak, and daily-driver evidence.
- [ ] Every result records exact build/environment/configuration/artifacts; later changes receive impact analysis and rerun every plausibly affected gate.
- [ ] No Severity 1/2 ships; Severity 3 requires a safe workaround, bounded scope, owner, and follow-up; any explicit hard-gate failure blocks regardless of severity.
- [ ] Any external browser/OS waiver documents the defect and safe fallback and weakens no correctness, security, authority, integrity, or recovery guarantee.
- [ ] Promotion evidence states accessibility and rich observability boundaries accurately and makes no unsupported WCAG, telemetry, penetration-test, or worker-sandbox claim.
